# Ideal Customer Profile (ICP)

> **최종 업데이트:** 2026-04-30

---

## ICP ≠ Persona

PostHog 프레임워크에서 ICP는 **고객 유형**(어떤 사람), Persona는 **개인의 동기와 행동 패턴**을 정의한다. Agentic30은 B2C이므로 ICP가 곧 이상적 개인 고객 세그먼트.

---

## Creator-first ICP

현재 Mac 앱은 **pre-dogfood 단계**다. Creator 본인도 아직 daily use 못 하고 있다. 이 시점의 "1차 고객"은 외부 유저가 아닌 **creator 본인**이다.

외부 유저 대상 ICP는 여전히 유효하지만, Q2 2026 제품 결정의 기준은:
1. 1차 — Creator 본인 (dogfood validator)
2. 2차 — 아래 정의된 외부 ICP (Q3+ 모집 대상)

Creator 본인도 아래 외부 ICP의 기준을 만족하므로, 같은 제약이 적용된다.

## Our ICP: 전업 1인 개발자 (수익 0원, macOS)

에이전트 코딩 도구(Claude Code, Codex)로 프로덕트를 만들 수 있는, 이미 전업한 1인 개발자로 아직 수익 0원인 사람. **macOS 사용자**이며, 프로젝트 path를 지정하고 업무 일지, 고객 인터뷰, Build in Public 기록을 꾸준히 남길 의향이 있다.

| | 전업 1인 개발자 (수익 0원, macOS) |
|---|---|
| **설명** | 퇴사 후 전업했지만 아직 단 1원도 벌지 못한 상태. 프로덕트를 만들 기술은 있지만 "뭘 만들어야 팔리는지" 모른다. 만들어도 유저가 오지 않는다. N번의 실패 후 "방법이 틀렸다"고 자각한 사람. |
| **기준** | - 이미 전업 (퇴사 완료, 저축 소진 중)<br>- 에이전트 코딩 도구 주력 사용 (Claude Code, Codex, Cursor)<br>- 수익 0원 (아직 첫 매출 없음)<br>- **macOS 사용자** (Intel 또는 Apple Silicon)<br>- **프로젝트 path 지정 가능**<br>- **업무 일지 작성 의향** (오늘 만든 것, 막힌 것, 배운 것)<br>- **고객 인터뷰 실행 의향** (Zoom, caret.so, 오프라인 녹음 등)<br>- **BIP 기록 의향** (Threads/블로그 게시글, 팔로워 수, 반응, 배움)<br>- 인터뷰 transcript를 파일로 확보 가능 |
| **왜 중요한가** | - **절박한 동기:** 저축을 태우고 있어 30일 스프린트에 진심<br>- **기술력 보유:** 만들기는 되므로 "무엇을 팔지"에 집중 가능<br>- **실행 데이터:** 프로젝트, 업무 일지, 인터뷰, BIP 기록으로 Adaptive 엔진 입력을 직접 생성할 수 있음<br>- **입소문:** 0→1 성공 사례가 개발자 커뮤니티에서 강력한 증명<br>- **재참여:** 첫 PMF 달성 → 다음 프로젝트에도 재참여 |
| **트리거** | N번째 프로덕트 실패 후 "내 방법이 틀렸다" 자각. 혼자 삽질(YouTube, 블로그)의 한계를 느낀 시점. |
| **현재 대안** | 혼자 삽질 — YouTube 튜토리얼, 블로그, 인디해커 콘텐츠 소비. 구조화된 대안이 없어 표류 중. AI 코파운더 SaaS는 많지만 내 프로젝트와 실행 기록 기반으로 맞춤 과제 주는 도구는 없음. |

## Needs / Haves / Don't Needs

### 필요한 것 (Needs)

- **수요검증 프레임워크** — 뭘 만들어야 팔리는지 찾는 체계적 방법 (Mom Test, 랜딩 검증, 경쟁 분석)
- **개인화된 실행 가이드** — "Day 2엔 Mom Test"는 누구나 알지만, 내 프로젝트와 기록에서 뽑아낸 내 상황 맞춤 질문이 필요
- **마케팅/유저 획득 능력** — 만들 줄은 알지만 사람을 데려올 줄 모른다
- **구조화된 30일 스프린트** — 마감 없이 표류하지 않을 데드라인
- **혼자의 고립감 해소 (후순위)** — 동료나 멘토의 존재. MVP에는 없음, 이후 검토
- **30일 안에 PMF 검증 결과** — 100명 + 첫 매출 같은 강한 신호를 목표로 하되, 미달 시에도 계속/전환/중단 판단을 근거로 남김

### 이미 가진 것 (Haves)

- AI 에이전트 코딩 능력 (Claude Code/Codex로 프로덕트 빌드 가능)
- 전업 의지 + 풀타임 시간
- macOS 장비
- 인터뷰 수행 의향 + 도구(Zoom, caret.so 등)
- 업무 일지와 BIP 기록을 남길 채널(Threads, 블로그, 노트 등)

### 필요하지 않은 것 (Don't Needs)

- 코딩 기초 교육 — 이미 AI 도구로 개발 가능
- 팀 매칭/공동 창업 지원 — 1인 개발자 포커스
- 투자 연결 — 부트스트래핑 우선
- 장기 커리큘럼 (6개월+) — 30일 집중 스프린트
- Windows/Linux 네이티브 지원 — MVP는 macOS 전용

## Anti-ICP (비타겟)

| 비타겟 | 이유 |
|---|---|
| 직장인 (퇴사 전 사이드프로젝트 단계) | 전업자의 절박함과 풀타임 시간이 없다. 30일 스프린트 강도 소화 어려움. |
| 이미 월 매출 있는 1인 개발자 | PMF를 이미 찾았다. Foundation 단계가 불필요. 성장 단계 프로그램이 적합. |
| 팀 기반 스타트업 희망자 | 1인 개발자 특화. 공동 창업이 목표라면 다른 프로그램이 적합. |
| Windows/Linux 전용 사용자 | Mac 앱 제품. 크로스 플랫폼은 MVP out of scope. |
| 기록 비의향자 | Adaptive 엔진은 프로젝트/업무 일지/인터뷰/BIP 기록이 핵심 입력이다. 기록 없이는 정적 커리큘럼 대비 이점 없음. |
| "대신 해줘" 수요자 | 30일 스프린트는 본인이 실행. 에이전트가 인터뷰하고 빌드하고 마케팅하는 솔루션을 원하는 사람은 다른 제품을 찾아야 함. |

## Persona

### Primary (유일): 퇴사 후 전업자, macOS 사용자

- **프로필:** 퇴사 완료, 저축 소진 중, 에이전트 코딩 도구로 프로덕트 빌드 가능, Mac 장비
- **핵심 동기:** N번 만들었는데 다 실패. "방법이 틀렸다"는 자각
- **핵심 고민:** "만들 줄은 아는데, 뭘 만들어야 팔리지? 어떻게 사람을 데려오지?"
- **인터뷰 경험:** 이미 고객 인터뷰 몇 번 해봤지만 정리/활용에 한계를 느낌
- **성공 기준:** 30일 후 실제 사용자 증거와 ask 결과로 계속/전환/중단 판단. 강한 성공 신호는 가입/사용자 100명 + 첫 매출(금액 무관).

## Downstream Decisions

ICP가 모든 전략적 의사결정을 결정한다 (PostHog: "Your entire strategy is downstream of your ICP"):

| 영역 | ICP 기반 결정 | 근거 |
|---|---|---|
| **배포** | macOS DMG 직접 배포 | ICP가 macOS 유저. App Store는 workspace 접근 제약으로 MVP 제외. |
| **가격** | 미정 (creator dogfood 이후 결정) | 외부 유저 모집 전 dogfood 검증 필수. 가격은 PMF 증명 이후. |
| **마케팅 채널** | 개발자 커뮤니티 (Threads, Discord), Claude Code 생태계 — Q3+ | 현재 외부 모집 non-goal. Creator BIP로 소셜 프루프 축적이 선행. |
| **콘텐츠 톤** | 기술적이되 실행 중심. 코드/터미널 친화 | macOS 메뉴바 앱 유저는 터미널 친숙, 이론보다 실행 명령 선호. |
| **기능 우선순위** | Adaptive 엔진 > Sub-workflow 도구 > UI 완성도 > 다중 기기 | ICP는 결과 중심. 예쁜 UI보다 맞춤 과제가 중요. |
| **온보딩** | Node/provider 셋업 → 프로젝트 path 선택 → 첫 업무 일지/인터뷰/BIP 기록 연결 | ICP가 설치/셋업 감수 가능. 일반 사용자 대비 기술 설정 높음. |
| **포지셔닝** | "내 프로젝트와 실행 기록으로 내 30일 커리큘럼을 만드는 macOS 어시스턴트" | "온라인 강의" 아님. "정보 플랫폼"도 아님. 개인화된 실행 시스템. |
| **UI 스타일** | 메뉴바 + 작은 패널, 시스템 네이티브 느낌 | 개발자는 OS 네이티브 앱 선호. 웹앱 같은 Electron 티 최소화. |

## ICP 검증 신호

### 긍정 신호

- 프로젝트 path를 지정하고 업무 일지/인터뷰/BIP 기록을 누적
- 인터뷰 transcript를 업로드하고 분석 결과에 질문/수정 요청
- Adaptive 과제를 실제 수행하고 결과를 다시 앱에 제출
- Mac 앱 관련 기술 피드백을 구체적으로 줌 (권한, 단축키, 통합 요청)
- Day 30 완주 + 본인 프로덕트로 실제 매출 발생

### 경고 신호

- "언젠가 해볼게요" — 긴급성 없음 (전업자가 아닐 수 있음)
- "Windows 버전 언제 나오나요?" — 잘못된 플랫폼
- "기록 없이 자동으로 해주면 안 되나요?" — 핵심 입력 수행 의지 없음
- "강의 영상은 어디 있나요?" — 정적 커리큘럼 기대
- "이미 매출은 좀 나는데..." — 이미 PMF 찾은 사람
