# Values — Agentic30이 지키는 기준

> **최종 업데이트:** 2026-05-22

Agentic30의 가치는 좋은 말이 아니라 제품과 코칭이 매일 지킬 판단 기준이다.

대상은 전업 1인 개발자다. 목표는 30일 안에 사용자 100명과 첫 매출 가능성을 검증하는 것이다. 그래서 Agentic30은 강의가 아니라, 사용자의 프로젝트와 기록을 읽고 다음 행동을 바꾸는 macOS assistant여야 한다.

---

## 1. 사용자가 드라이브한다

사용자의 프로젝트, 인터뷰, 업무 일지, BIP 기록이 커리큘럼을 바꾼다.

- 우선: 실제 기록 기반 다음 행동
- 거절: 고정 강의, 범용 조언, 대신 실행해주는 agency

## 2. 증거는 끌어오고, 밀어붙이지 않는다

관심, 칭찬, 조회수는 수요가 아니다. 행동, 반복 사용, 결제 의사, 첫 매출이 증거다.

- 우선: 외부 ICP 인터뷰와 private pilot feedback
- 거절: “좋아 보인다”, “언젠가 쓸게요”를 성공 신호로 보는 것

## 3. 투명하게 말한다

Agentic30은 사용자를 기분 좋게 만드는 답보다, 지금 멈추거나 바꿔야 할 이유를 명확히 말한다.

- 우선: 근거, 반증, 다음 검증 행동
- 거절: 모호한 응원, 과장된 확신, 숨은 가정

## 4. 쉽게 만든다

전업 1인 개발자는 이미 만들 줄 안다. 필요한 것은 복잡한 시스템이 아니라 오늘 바로 실행할 수 있는 좁은 행동이다.

- 우선: 설치 후 빠른 첫 과제, 짧은 질문, 명확한 출력
- 거절: 긴 커리큘럼 설명, 복잡한 온보딩, 불필요한 설정

## 5. 명료함이 에고보다 중요하다

아이디어를 지키는 것보다, 고객 증거에 맞게 좁히고 바꾸는 것이 중요하다.

- 우선: continue / pivot / stop 판단
- 거절: 만든 것이 아까워서 계속 만드는 것

## 6. 혼자 만들지만 고립되지 않는다

1인 개발자는 혼자 실행하지만, 혼자 판단하면 편향된다. Agentic30은 고객 목소리와 공개 기록을 판단에 끌어들인다.

- 우선: 인터뷰, BIP, 사용자 피드백, 다양한 반응
- 거절: 혼자 세운 TODO만으로 제품 방향을 결정하는 것
